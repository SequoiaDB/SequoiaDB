BEAM for Windows

BEAM has been ported to run natively in a Windows environment.

We consider the current release to be Beta because it has not received an equal amount of testing as Linux/AIX. Getting beta testers has proven to be a very difficult endeavor.

Here is a quick fact sheet:

  • BEAM for Windows was built on Windows 2000 and should run on all Windows versions newer than that. Our nightly regression buckets run on XP and that works just fine.
  • The languages C, C++, and Java are supported.
  • The BEAM Ant Task is supported and installed with BEAM.
  • The BEAM Eclipse Plugin is supported and available from our web site.
  • BEAM ships with some Perl scripts in the bin directory. These scripts require a Perl interpreter in order to properly function. If you don't have Perl installed, visit the ActivePerl page and download Perl from there. ActivePerl is a Perl distribution provided by ActiveState for no charge.

    Note that while some scripts (like beam_configure) will work with MSYS Perl, BEAM in general does not support it. BEAM requires a Windows-flavored Perl.

    Once you download and install ActivePerl, the Perl program will be in your PATH. You can run any BEAM Perl program by prefixing the BEAM command with perl as you run it from the Windows command console, like so: 

    perl beam\bin\beam_configure --c gcc -o gcc.tcl
perl beam\bin\beam_sort_log a.complaints a.sorted
perl beam\bin\beam_diff_log --common a.complaints b.complaints
perl beam\bin\beam_stats a.stats

  • Compiler emulation for Microsoft Visual-C++ version 8 is provided. See this page for more details.
  • Compiler emulation for IBM VisualAge for Windows is provided. See this page for more details.
