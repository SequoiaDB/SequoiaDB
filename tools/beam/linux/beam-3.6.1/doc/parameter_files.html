BEAM Parameter Files

Introduction

BEAM ships with a few different parameter files. These are pre-canned settings that form various starting points for customizing BEAM for your project.

Parameter Files

beam_default_parms.tcl
This is the default set of parameters. If you don't specify a parameter file on the beam_compile command line, this is the one that is used.

This enables some of the BEAM complaints that we think are the most useful for a wide audience. It will be enough to get you started, but you will want to customize BEAM to get the most from it.

Use to begin a custom file: 

source beam_default_parms.tcl
...

beam_level1_parms.tcl
This file enables more checks than beam_default_parms.tcl. It is useful for projects that want to turn up the heat a little bit. Once your code is "BEAM clean" with the default parameters and you are interested in a deeper analysis, give this setting a try.

It might run a little longer and might produce a few more false complaints but the trade-off is that it has been designed to help find more elusive bugs. Some of the complaints may not make sense for your project; in that case, start a custom file with beam_level1_parms.tcl and disable those complaints that you don't want to see.

It is called "level1" because we plan on adding more levels in the future for even deeper analysis.

Note: This file is only useful for C, C++, and Java. For other languages, it is no different from beam_default_parms.tcl.

Can be used on the command-line: 

$ beam_compile --beam::parms=beam_level1_parms.tcl ...

or to begin a custom file: 

source beam_level1_parms.tcl
...

beam_never_parms.tcl
This file will start you off with no complaints enabled. You would use this when you want to start with a clean slate and enable a specific set of checks.

Use to begin a custom file: 

source beam_never_parms.tcl
...

beam_always_parms.tcl
This file will start you off with every complaint enabled. You would start with this and remove the complaints that you did not want to see.

Please understand that this file is not a parameter setting that will produce the largest amount of usable complaints. That is neither the case nor the intent. It is only meant to seed your own configuration. Using is standalone without additional configuration is a rotten idea. You have been warned.

Use to begin a custom file: 

source beam_always_parms.tcl
...

IMPORTANT

The file tcl/beam_default_c_and_cpp_parms.tcl contains all possible settings for all parameters. Many are commented out, but they are there just to show you the possibilities. Therefore copy and paste from that file when setting up your own parameters.

Each new release of BEAM contains new checks. We enable every new check in beam_always_parms.tcl, but only those we consider useful for everybody get enabled in beam_default_c_and_cpp_parms.tcl. Thus if your own Tcl script starts by sourcing beam_always_parms.tcl then each release will give you new complaints, which you may have to disable. On the other hand, if your Tcl script starts by sourcing beam_default_parms.tcl then you need to try new checks to see if you want to enable them.
