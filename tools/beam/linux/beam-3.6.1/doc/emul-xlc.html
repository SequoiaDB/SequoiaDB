Details of xlc / vac Emulation

The current level of xlc/vac emulation has been proven to be robust enough for a wide variety of application programs as well as system related code.

Command-line Flags

Please see the file tcl/beam_map_vac_flags.tcl or beam_map_xlc_flags.tcl in your BEAM installation.

beam_map_xlc_flags is for the older IBM compilers (version 4 and under). beam_map_vac_flags is for the newer compilers, and has been tested with VisualAge 5, 6, 7, and 8.

Predefined Macros

The list of predefined macros can be found in the compiler configuration file that beam_configure generated for you.

Additional Keywords and Syntax Extensions

The IBM compilers support additional keywords and syntax extensions in source files. The following keywords are recognized and will be ignored if the compiler supports them. More specifically, BEAM will create a macro under the covers that replaces the keyword with a sequence of blanks.

__cdecl _Cdecl
__export _Export
__far16 _Far16
__far32 _Far32
__fastcall _Fastcall
__import _Import
__inline _Inline
__optlink _Optlink
__packed _Packed
__pascal _Pascal
__seg16 _Seg16
__stdcall _Stdcall
__system _System
__unaligned _Unaligned

Configuration Files

If no -F flag was given, we attempt to open the default configuration file as follows:

Compiler Configuration file Stanza
xlc (ibmcxx) and C file /etc/ibmcxx.cfg xlc
xlc (ibmcxx) and C++ file /etc/ibmcxx.cfg xlC
xlc (VisualAge) and C file /etc/vac.cfg xlc
xlc (VisualAge) and C++ file /etc/vac.cfg xlC

If your configuration file resides in a non standard location, you need to specify it on the command line using the -F flag.
