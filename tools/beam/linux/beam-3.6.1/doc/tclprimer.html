Tcl Primer

This document summarizes those few facts that you need to know about Tcl syntax.

The statement: 

set parm val

assigns the string val to the variable parm. The contents of parm can then be retrieved by the expression $parm. Like this: 

set parm2 $parm

Now parm2 holds the same value as parm, namely val.

The statement: 

lappend parm s1 s2

appends the two strings s1 and s2 to the list parm. To erase the existing contents of a variable parm use the command: 

set parm ""

Both variable names and values may be enclosed in double quotes or braces. (There is a difference between double quotes and braces, which we will not go into.) This is the way you can have variable names or values containing blanks. For example, 

lappend "x y" {a b} c

operates on the variable x y, whose name contains a blank, and it appends two strings -- a b and c.

Variable names can also indicate array notation, as in 

set "a(x, y)"  "v"

Please beware that blanks are significant. The above is different from 

set "a(x,y)"  "v"

A namespace is a set of variables sharing a prefix of the form abc::x, abc::y, etc. A namespace abc can be introduced with the command 

namespace eval abc {}

but in BEAM, namespaces automatically come to life when they are used.

For example, 

namespace eval abc {
    set     x v
    lappend y $x
}

Has the same effect as 

set      abc::x v
lappend  abc::y $abc::x

and we recommend the latter because it is clear where every variable lives.
