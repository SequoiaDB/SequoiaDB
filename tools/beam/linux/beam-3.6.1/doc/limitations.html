Known Limitations of the EDG Parser

Currently, the following limitations exist.

  • C++ code that includes the GNU C++ header <complex> or <complex.h>, or that uses complex types directly, will cause parse errors. Note that C code may use complex types.
  • Taking the address of a bitfield is unsupported in most cases. For this reason, using offsetof() with XLC on bitfield members will fail, even though XLC allows it.
  • In GCC compatibility mode: Code which uses GCC built-in functions (like memset) without a declaration in scope, and which relies on the function type being known, will fail to compile. Always include a declaration to avoid this problem. We have seen this in the Linux kernel source.
