Download and Installation Instructions

The BEAM distribution contains all executables, libraries, and configuration scripts that are necessary to run BEAM.

  1. License
    BEAM is licensed for IBM internal use only. That means: BEAM can only be installed on IBM-owned hardware and only regular IBM employees and IBM contractors may run BEAM. By downloading BEAM you acknowledge and agree to this license.

  2. Download
    Click on one of these links.

    Note: If you are downloading one of our non-Windows packages from Internet Explorer, it may save the file as .tar.tar. You will have to rename it to .tar.gz before you will be able to extract it. This seems to be a Windows or Internet Explorer bug.

    GNU/Linux for x86
    (glibc 2.3.2 or higher)
    GNU/Linux for s390x
    (glibc 2.3.3 or higher)
    Windows 2000 and up
    z/os release 4 and up

    Save the file to disk. Depending on which browser you use, you might have to press the "shift" key while clicking on the link.

  3. Extract
    Enter one of the following commands depending on your OS:

    OS Command
    AIX gunzip -c beam-3.6.1.tar.gz | tar pxf -
    GNU/Linux gunzip -c beam-3.6.1.tar.gz | tar pxf -
    Windows Execute the installer by double-clicking on beam-3.6.1-install.exe
    USS gunzip -c beam-3.6.1.tar.gz | pax -rv -pp

    This will extract the BEAM distribution into a single directory beam-3.6.1. If you need gunzip and don't have it, you can get get it from http://www.gzip.org.

  4. File permissions
    If you are installing BEAM in one of the afs/dfs/gsa file systems, make sure that the access control lists for the directory containing the BEAM installation are set appropriately.

  5. Verify
    Save this C file to disk and then invoke BEAM using the beam_compile from your installation's bin directory like this: 

    beam-3.6.1/bin/beam_compile -c example.c

    You should see these complaints.

  6. Install point overview

    There are several subdirectories. For BEAM users only the bin, doc, and tcl directories are of interest.

    • bin
      BEAM binaries and scripts. The ones that might interest you are:
      • beam_compile
        The BEAM driver. Invokes the corresponding BEAM frontend on one or more source files.
      • beam_configure
        Figures out compiler and target machine specifics and write them to a configuration file.
      • beam_diff_log
        Compute differences between two file with BEAM complaints.
      • beam_sort_log
        Sort a file containing BEAM complaints.
      • beam_stats
        Analyzes a stats file.
      • beam_trace
        Watches a build and invokes BEAM alongside the compiler.
    • doc
      A copy of BEAM's online documentation. For those who like to grep for things.
    • tcl
      Tcl configuration scripts. You will need to refer to some of the scripts in this directory if you want to customize BEAM.

    All other subdirectories are for BEAM's internal organization only and are subject to change without notice.

  7. Leave it alone
    Never modify any of the files of the BEAM distribution. Doing so makes it very hard to recreate a problem that you might encounter.
