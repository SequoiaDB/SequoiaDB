The BEAM Tcl Search Path

Whenever a Tcl file is loaded, it has to be located. Loading happens as a side effect of using source file from within a Tcl script, or --beam::source= or --beam::parms= or other similar command-line options that specify Tcl files.

If the file that is given is an absolute path name (it starts with "/"), then no searching is done, and the named file is loaded directly.

If the file that is given is a relative path that contains "/" anywhere in it, then no searching is done, and the named file is loaded directly, relative to the current working directory.

If the file is a plain file name with no "/" anywhere in it, then it is looked for in the following directories, in the following order. The first file found is loaded.

  • The current working directory
  • The tcl subdirectory of the BEAM installation

Examples: 

# Load a file explicitly from a parms.tcl file:

source /home/foo/file.tcl

# Search for the file in the BEAM search path:

beam_compile --beam::source=file.tcl ...
