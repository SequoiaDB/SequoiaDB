What is BEAM?

BEAM is a static analysis tool that looks for bugs in C, C++, and Java software. That is, it does not try to execute the source code and therefore does not require a testcase. It analyzes the software in a fashion similar to a compiler, lint, or a formal verifier. It differentiates itself from the others by making an effort to decide which potential problems are really feasible and report only the feasible ones.

Like other such tools the problems it reports include bad memory accesses (uninitialized variables, dereferencing null pointers, etc), memory leaks, unnecessary computations, and so on. It can also be given extra information to detect application-specific errors concerning locking, data base accesses, etc.

Unlike other static analysis tools, BEAM is able to report more real errors without overwhelming the user with bogus ones. It performs the necessary theorem proving to decide feasibility of paths and can therefore do a better job of identifying real problems. As a result, BEAM reports more problems than a compiler or lint would while avoiding false ones.

However, deciding feasibility of paths is a computationally hard problem which requires exponential resource (in the general case). BEAM prefers to spend time (rather than blowing up memory) to solve this problem. You should adjust your expectations accordingly. As a ball-park figure, BEAM takes 7 CPU hours to analyze 1.4 million lines of code on a 2GHz Pentium with 1GB RAM. For modern, well-structured code (i.e. not spaghetti code) it is often possible to run BEAM interactively. However, path analysis is data dependent, so your mileage may vary. You can limit the amount of time spent by trading off the number of reported problems.

BEAM uses some of the techniques found in formal verifiers, but does not require any specifications (which prevents it from proving actual functionality). Unlike verifiers, which try to form a proof of correctness and issue a complaint if they fail, BEAM tries to form a proof of incorrectness and issues a complaint if it succeeds.
