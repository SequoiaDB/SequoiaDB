Details of ICC Emulation

Intel's ICC compiler uses the EDG parser for C/C++ processing (just as BEAM does). Therefore, the main emulation that BEAM provides is in translating ICC's command-line flags.

Command-line Flags

Please see the file beam_map_icc_flags.tcl for the list of supported flags. That list was derived from ICC version 9.0. If you find that a flag that changes compiler properties is not supported please let us know.

-Klang
This option is partially supported. Only one -K option can be specified. Independent of where it appears on the command line, it will apply to all source files.

-gcc-version=version
This option cannot be handled by the command line mapper as it changes too many things. Therefore, you need to generate a separate compiler configuration file for each value of the -gcc-version flag. 

beam_configure --c icc -o icc-320.tcl -- -gcc-version=320
beam_configure --c icc -o icc-330.tcl -- -gcc-version=330

And later, when invoking BEAM, specify the configuration file that matches the -gcc-version flag. 

beam_compile --beam::compiler=icc-320.tcl -c -gnu-version=320 foo.c
beam_compile --beam::compiler=icc-330.tcl -c -gnu-version=330 bar.c

Predefined Macros

The list of predefined macros can be found in the compiler configuration file that was generated by beam_configure.
