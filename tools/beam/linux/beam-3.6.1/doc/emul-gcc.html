Details of gcc Emulation

Because EDG provides robust support for GCC/G++ extensions and features, the main emulation that BEAM provides is in translating the command-line flags from the GCC/G++ command line to EDG options and compiler configuration options.

Command-line Flags

Please see the file beam_map_gcc_flags.tcl in your BEAM installation.

Notes:

-m32 and -m64
What these flags do (for GCC) is to define a target machine with different characteristics. In BEAM the same thing is achieved by specifying the corresponding compiler configuration file. Which is what you need to do. You can still pass the option but it will be ignored. However, BEAM will issue a warning if you specify a configuration file that seems wrong for the given option.

You can easily generate separate compiler configuration files for 32-bit for 64-bit targets by invoking beam_configure like so: 

beam_configure --c gcc -o gcc32.tcl -- -m32
beam_configure --c gcc -o gcc64.tcl -- -m64

-x lang
This option is partially supported. Only one -x option can be specified. Independent of where it appears on the command line, it will apply to all source files.

Predefined Macros

For gcc and g++, the predefined macros that will be used are contained in the compiler configuration file that is passed to BEAM.
