Details of sCC2010 Emulation

The current level of sCC2010 emulation is mature. We have been able to work around most language extensions that sCC provides. Experiments with PHYP firmware show that BEAM can parse close to 100% of C++ code written for sCC.

Target Machine

The target machine has the following characteristics:
  • 8 bits per byte
  • big endian
  • a plain char is unsigned
  • a plain bitfield is unsigned
  • wchar_t is unsigned

The built-in data types are implemented according to this table:

Datatype Sizeof Alignment
char 1 1
short 2 2
__int32 4 4
int 8 8
long 8 8
long long 8 8
float 4 4
double 8 8
long double 8 8
size_t 8 8
wchar_t 2 2
pointer 8 8

Predefined Macros

The list of predefined macros can be found in the files tcl/beam_sCC2010_cpp_config.tcl in your BEAM installation.

Command Line Flags

Please see the file tcl/beam_map_sCC2010_flags.tcl in your BEAM installation.

File Extensions

The following file extensions are recognized:
  • .C
  • .cpp
  • .c only in combination with the -+ command line flag
